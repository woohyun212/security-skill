## Report Generation Logic

The inspection session writes results to `/tmp/isms_check_<YYYYMMDD_HHMMSS>.md`. The report contains four sections:

### Report Header

Fields written at session start:
- Organization name
- Certification scope (or "Not specified")
- Inspection date
- Domain selection

### Inspection Results Summary Table

| Column | Description |
|--------|-------------|
| Compliant | Count and percentage of `y` answers |
| Non-compliant | Count and percentage of `n` answers |
| Not applicable | Count of `s` answers |
| Total | Sum of all inspected controls |

Compliance rate formula: `Compliant / (Compliant + Non-compliant) * 100`

### Control Item Status Table

Full per-control listing with columns: Control ID, Item name, Domain, Status icon (✓ / ✗ / -).

### Gap Analysis and Remediation Recommendations

Non-compliant items grouped by domain. Priority tiers applied automatically:

| Priority | Timeframe | Scope |
|----------|-----------|-------|
| HIGH (Immediate) | Now | Personal information collection/processing, access control failures |
| MEDIUM (Short-term) | Weeks | Personnel security, physical security, cryptography |
| LOW (Mid-to-long-term) | Months | Management system documentation, training, disaster recovery |
